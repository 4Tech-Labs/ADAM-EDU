## Architecture Decisions
- keep `queryKeys.student.caseDetail(assignmentId)` as the canonical runtime query key and preserve `case()` only as a deprecated alias
- keep autosave orchestration in the hook, not the page, so debounce, retry, reconciliation, and unmount flush stay bound to state ownership
- treat backend sanitization and structured error envelopes as authoritative for student detail, draft, and submit endpoints
- reuse the shared case-viewer surface for student and teacher flows by extracting `PreguntaCard`, `SectionRail`, and `SolucionEsperadaRenderer`
- update `student.cases` and `student.courses` cache optimistically on submit success so the dashboard never flashes stale actionable state
